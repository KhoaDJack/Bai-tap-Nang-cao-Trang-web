📋 API Overview

Base URL: http://localhost:3000/api

Version: 1.0.0 (No versioning yet)

Format: JSON

Authentication: ⚠️ None (Security concern)

⚠️ Current Status: Development/Demo only. Not production-ready.

Response Format

{ "data": {...}, "error": "error message" (only if error) }

HTTP Status Codes

Code Meaning
200 OK - Request succeeded
201 Created - Resource created successfully
400 Bad Request - Invalid parameters
404 Not Found - Resource not found

🏠 Rooms

GET Get All Rooms

GET /api/rooms

Retrieve a list of all available hotel rooms.

Response Example (200 OK)

[ { "id": 1, "title": "Standard Room", "description": "Cozy and bright room with queen bed", "price": 110, "capacity": 2, "slug": "standard" } ]

⭐ Reviews

GET Get All Reviews

GET /api/reviews

POST Create Review

POST /api/reviews

Request Body

Field Type Status Example
stars integer Required 5
title string Required "Great Stay"
message string Required "Excellent service"
author string Required "John Doe"

PUT Update Review

PUT /api/reviews/{id}

DELETE Delete Review

DELETE /api/reviews/{id}

👥 Staff

Manage staff members (full CRUD operations available)

Method Endpoint Description
GET /api/staff List all staff
POST /api/staff Create new staff
PUT /api/staff/{id} Update staff
DELETE /api/staff/{id} Delete staff

🛎️ Services

Manage hotel services (Room Service, Concierge, etc.)

Method Endpoint Description
GET /api/services List all services
POST /api/services Create new service (name + description only)
PUT /api/services/{id} Update service
DELETE /api/services/{id} Delete service

📅 Bookings

GET Get All Bookings

GET /api/bookings

Retrieve all hotel room bookings.

POST Create Booking

POST /api/bookings

Request Body

Field Type Status
roomName string Required
roomType string Required
roomId integer Required
guestName string Required
guestId string Required
checkin date Required
checkout date Required
guests integer Required
nights integer Required
totalCost number Required

POST Check Availability

POST /api/availability

Check if a room type is available for given dates.

Request Body

Field Type Status
checkin date Required
checkout date Required
guests integer Required
roomId integer (1-5) Required
roomType string Required

Response Example

{ "available": true, "nights": 2, "totalCost": 220, "roomId": 1, "roomName": "Standard Room", "roomType": "standard", "message": "Standard Room is available for 2 night(s)..." }

PUT Update Booking

PUT /api/bookings/{id}

DELETE Delete Booking

DELETE /api/bookings/{id}

📊 Architecture Assessment

Overall Score: 5.6/10

Status: ⚠️ ACCEPTABLE for development/demo, but NOT production-ready

Strengths ✅

  • RESTful Design: Proper HTTP methods (GET, POST, PUT, DELETE)
  • Consistent URL Patterns: /api/{resource}/{id} format
  • Error Handling: Returns JSON errors with context
  • 6 Resource Modules: Rooms, Reviews, Staff, Services, Bookings, Availability

Weaknesses ⚠️

  • 🔴 CRITICAL: No API Documentation (this file is manual)
  • 🔴 CRITICAL: No Authentication/Authorization
  • 🔴 CRITICAL: No Input Validation
  • 🟠 HIGH: No API Versioning (/api/v1/)
  • 🟠 HIGH: File-based persistence (not scalable)
  • 🟠 MEDIUM: No request logging
  • No test suite
  • No rate limiting

Recommendations

  1. ✅ Install Swagger-UI for auto-generated docs
  2. ✅ Add JWT authentication
  3. ✅ Add input validation middleware
  4. ✅ Migrate to database (SQLite/MongoDB)
  5. ✅ Add API versioning
  6. ✅ Add unit tests
